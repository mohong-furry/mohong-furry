简介

名称定义

  • 中文名称：字符类数据库
  • 全称：文本比特类数据库 (TextBiteSQL)
  • 简称：TBSQL (T-Text 文本 | B-Bite 比特 | SQL-数据库)

用途

TBSQL是一种配置性类数据库，适用于：

  • 简单映射
  • 常数定义
  • 商品信息
  • 配置型数据存储

文件结构

  • 文件扩展名：.tbs
  • 组成：行标识（ids）和列定义（token）
  • 单表容量：0 - 1,000,000 行（仅支持正整数）

设计理念

  • 采用类数轴概念，区域参考阴影，ids轴需要翻转
  • 单表可表达数位于 0-10⁶个内（正整数）
  • 勿使用负数或小数防止处理异常
数轴示意图： 值(token) ↑ │ ┌───┬───┬───┐ │ │ │ │ │ │ ├───┼───┼───┤ │ │ 值 │ 值 │ 值 │ │ ├───┼───┼───┤ │ │ 值 │ 值 │ 值 │ │ └───┴───┴───┘ │ └──────────────────→ 键(ids)

数据模型

术语定义
  • ID: 唯一标识符
  • token: 值定义（列名）
  • ids: 键（行标识）
  • 值: 具体数据
实际存储结构
ID token₁ token₂ token₃ ids₁→┌───┬──────┬──────┬──────┐ │ │ 值 │ 值 │ 值 │ ids₂→├───┼──────┼──────┼──────┤ │ │ 值 │ 值 │ 值 │ └───┴──────┴──────┴──────┘
注意：第一行都是值定义，单表容量：0 - 1,000,000 行（仅支持正整数）

类型定义

数据类型

  • int：整数
  • float：小数
  • bool：布尔
  • STR：字符串类型（默认）

使用规则

  • 值定义可用 =/= 表达字符串类型
  • 字符串类型是STR，可以不写，遇到解析问题时可以写上去
  • 示例整数定义方式：123=/=int
  • 前表都是默认STR类型来解析

索引方案

  • 方式：逐行扫描，首字段匹配
  • 流程：
    1. 定位表头，确认ID列位置（首列）
    2. 读取数据行，检测第一个分号 ;
    3. 提取分号前字符串作为当前行ID
    4. 比较当前行ID与目标ID：
      • 匹配：完整读取该行，返回数据
      • 不匹配：立即终止当前行读取，跳转至下一行
    5. 重复直至找到目标或文件结束
优化：避免全行分割，仅解析首字段

文件组织架构

层级结构

  • TBE：单个 .tbs 文件（表）
  • TBZ：40个TBE为一组
  • TBP：40个TBZ为一堆
  • TO：全部TBP为一"奇"（奇点之意）

IDE文件系统

  • 使用 .ide 文件存储文件树内所有ID值以力键来搜索
  • 每个文件内不存在相同ID，所在的TO内也不存在相同ID
IDE文件演示内容： ID:1-99 PATH:TO UNAME:演示数据库 ID:1-999 PATH:TO UNAME:演示名

并发控制方案

原子操作保障

  • 写操作（add,rmrf,echo,move）采用原子化设计
  • 文件锁机制确保多进程操作数据一致性

锁文件设计

  • 每个.tbs表文件对应锁文件：{表名}.tbs.lock
  • 锁文件内容：进程ID:时间戳
示例：12345:2024-01-15T10:30:00

加锁规则

  • 写操作加锁：add,rmrf, move, echo (修改值)
  • 读操作共享：find,ls, lssign

锁超时机制

  • 最大锁等待：30秒
  • 锁超时时间：5分钟
  • 检查间隔：100毫秒

Service Registry 服务注册表

设计理念

自动管理的待办任务队列，任务完成后自动删除

表结构

ID;service_name;status;created_at;target_date;task_type

字段说明

  • ID:任务唯一标识
  • service_name:任务名称
  • status:状态（active|completed）
  • created_at:创建时间
  • target_date:计划完成日期
  • task_type:任务类型（file_operation|system_operation）

自动管理规则

  • 系统初始化时自动插入待办任务
  • 任务完成时自动标记为completed
  • 文件操作任务最长执行时间1小时
  • completed状态任务1小时后自动删除
  • 不提供 add/update/remove 命令

默认待办任务

1;concurrency_control;completed;2024-01-15;2024-02-01;file_operation 2;server_architecture;active;2024-01-15;2024-02-15;file_operation 3;storage_efficiency;active;2024-01-15;2024-03-01;system_operation

只读查询命令

  • TBS service status（查看当前待办任务状态）
  • TBS service completed（查看已完成任务历史）

模式定义

设计理念

查询时通过第一行的值定义获取类型，输出时值附带类型信息

模式格式

ID:INT;name:STR;age:INT;email:STR

输出规则

查询时返回带类型标记的值：

ID=1=/=INT;name=张三=/=STR;age=25=/=INT;email=zhangsan@email.com=/=STR

使用示例

user_info.tbs:

ID:INT;name:STR;age:INT;email:STR 1;张三;25;zhangsan@email.com 2;李四;30;lisi@email.com

查询输出：

TBS find user_info 1 输出：ID=1=/=INT;name=张三=/=STR;age=25=/=INT;email=zhangsan@email.com=/=STR

解析过程

  • 读取第一行获取每个列的类型
  • 查询数据时，为每个值附加类型标记
  • 输出格式：列名=值=/=类型
